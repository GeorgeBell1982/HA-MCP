# Test strategy

## Layers

1. Unit: schemas, paths/symlinks, redaction, risk/policy, proposal state/hash/expiry, YAML document edits, diffs, validation parsing, audit records, impact mapping, transaction state, Git scoping.
2. Integration: mocked HA REST and WebSocket servers covering auth, reads, config check, reload commands, response validation, correlation, reconnect, timeouts, safe retry, and no mutation retry.
3. Filesystem/Git: temporary directories and repositories only, realistic split HA fixtures, all include tags, `secrets.yaml`, comments, packages, dashboards, stale files, symlink attacks, dirty/conflicted Git states.
4. Workflow: inspect -> propose -> review -> validate -> apply -> reload -> verify -> commit, plus injected failure and rollback at every transition.
5. E2E: pinned Docker Home Assistant fixture when practical, no production dependency. Run read-only first; mutation uses disposable volumes and test credentials only.
6. MCP/CLI: stdio client smoke tests, tool inventory/annotations, structured envelopes, pagination, CLI exit codes, diagnostics, signal/cancellation behavior.

## Security gates

- Canary credentials must never appear in captured stdout/stderr, MCP content, audit JSONL, diffs, Git objects/messages, snapshots, or test reports. Tests include direct/renamed/aliased/symlinked secret-source reads, searches, diffs, diagnostics, errors, and audit paths.
- Adversarial path tests cover traversal, absolute/control/NFC/separator variants, malformed/oversized/NUL/invalid-UTF-8 protocol output, protected hard-link identity, source-count/identity bounds, cancellation byte cleanup, permit queue cancellation/deadline/handoff, and root identity changes.
- Linux native-helper tests must compile and run openat2 against final/parent symlinks, magic links, cross-mount paths, root swaps, same-size concurrent changes, cancellation, crash, and unsupported-kernel behavior. These active-race checks are UNVERIFIED on the current Windows host and remain a Slice G image gate.
- Windows asserts capability unavailable and must never substitute realpath/lstat as an active-race-safe implementation.
- YAML gate tests generate exact 512 KiB/N+1 byte and multibyte inputs, 100,000/N+1 structural counts, 64/N+1 syntax depth, 100/N+1 aliases/references/fanout, aggregate-reference limits, and expansion bombs. Fixtures cover empty/explicit-empty, LF/CRLF preservation, invalid encoding/BOM/NUL/CR, directives/documents, every allowed/denied tag, primitive/complex/duplicate/merge keys, duplicate/unresolved/cyclic anchors, surrogate chunk boundaries, bounded and sanitized unexpected diagnostics, and pre/mid-parse plus post-parse structural/expanded cancellation/deadline. No massive fixture blob is checked in.
- Repository-read tests cover canonical protected pre-denial, hard-link identity denial, digest freshness, replacement/same-inode edit/deletion/during-output drift and latching; catalog ordering/hashes, canonical identity-bound snapshot framing, catalog-root/file identity/size drift; file/content/search/public bounds; nontruncating redaction; linear KMP literal search with repeated-prefix exact-budget N/N+1 and cancellation, hidden-match removal, query-canary non-leakage, and Unicode/UTF-8 clipping; cursor invalid_input classification for malformed/auth/query/operation/rotation/closure and stale_source for authenticated snapshot/offset drift without health latching; cancellation; and sanitized failures.
- Native catalog protocol tests cover framing, zero-reserved header/record fields, version/type/count/path/UTF-8/NFC/global-order/duplicate/type-size corruption, directory metadata drift, pinned-O_PATH/readable-root traversal, bounded process behavior, two-pass equality, and unsupported capability. Linux compilation and active-race execution remain UNVERIFIED until the Slice G clean candidate-image gate.
- Slice E projection tests cover unchanged strict metadata; selective-IR freeze/surface; all include tag shapes, UTF-8 ordering, nested relative normalization, packages, empty semantics, repetition, cycles, missing/wrong-kind/escape/protected alias; automation/scene/script/helper identity forms and duplicates; all blueprint domains and isolation; plain/quoted/multiple !secret source-span masking; cursor operation/type/snapshot binding; root/file/content drift; sanitized errors; and inclusive file/edge/depth/work/node/resource/string/aggregate-source-byte/ID boundaries; explicit empty merge no-ops versus no-document failures; repeated-directory work; directory-index, offset, secret-range, finish/snapshot cancellation boundaries; high-cardinality irrelevant root/package/domain maps, automation non-ID keys, blueprint metadata cancellation/deadline polling; and immutable deterministic type-index paging plus exact-key get semantics.
- Tool inventory asserts absence of generic shell, arbitrary file write, arbitrary service call, and delete tools.
- Mutation defaults are tested from an empty environment.
- Audit partial writes/concurrent appends/permissions/rotation failure, stale proposal, forged/replayed/expired/wrong-digest approval, server restart, unrelated Git dirt, and failed rollback must fail closed.
- Auth tests reject userinfo/unsupported schemes and prove bearer headers are not sent on redirects, cross-origin destinations, or independently supplied WebSocket URLs.
- Recent-error tests cover authenticated WebSocket connection reuse/reconnect, command failure, oversized messages, malformed structured entries, timestamp/count/source validation, token/webhook redaction, exception exclusion, and bounded summaries.
- Add-on manifest tests assert Phase 1 has no `/config`, Supervisor, Docker, privileged, host-network, or host-filesystem capability and declares only the Core API permission needed.
- HTTP tests cover disabled/unprovisioned startup, ingress-only per-client 256-bit pairing, hash persistence/permissions, constant-time bearer verification, query rejection, per-client/rotate-all revocation, access-log canaries, Host/Origin/DNS rebinding, trusted/untrusted forwarded headers, request/header/output/time/session/concurrency/rate bounds, graceful overload, distinct-client session ownership, expiry/replay/reconnect, and cross-client rejection.
- TLS tests cover ECDSA certificate/key generation, mode-restricted storage, fingerprint rendering, expiry/corruption, recoverable rotation, all-session invalidation, non-loopback plaintext server/client refusal, passive/active MITM rejection, and certificate-rotation recovery without TOFU bypass.
- Bridge tests cover OS credential-store/fallback behavior, per-client credential redaction, pinned certificate fingerprint, expected server identity and compatible version pinning, mandatory re-pairing after credential/certificate/identity change, and malicious/oversized remote responses. They prove bearer credentials are never transmitted to non-loopback `http://` endpoints.
- HA credential tests prove the injected runtime token is used only for fixed Core proxy HTTP/WebSocket paths and never appears in options, diagnostics, audit, errors, logs, MCP results, HTTP/bridge traffic, snapshots, or fixtures.

## Platform matrix

The Home Assistant add-on `aarch64` image is the authoritative Phase 1 artifact. Linux/amd64 container runs development and clean-room tests where practical. Windows runs platform-neutral core and bridge tests with platform-specific credential/path cases; macOS is a best-effort core/bridge lane. Deployment adapters use contract tests across supported lanes and native HA OS E2E only in a disposable environment where available.

## Phase gates

Slice F Windows-neutral tests use a fake broker and native source inspection to exhaust fixed operation/argv/environment/protocol/stdin/output, abort/kill/sanitization, forbidden Git/network/write behavior, strict status/plumbing parsers, deterministic internal patches, born/unborn/detached and SHA-1/SHA-256 repositories, staged/unstaged/untracked/both semantics, rename-as-delete/add omission, conflicts/deletions/binary/non-YAML/symlink/gitlink/submodule denial, malformed paths and N+1 bounds, root/Git/index/object/current/protected drift, secret names/values/digests, truncation/hash semantics, and cancellation/deadline. Windows-neutral closure contracts require exactly one absolute runtime loader, no more than 16 absolute unique library paths, fixed ordering, and rejection of missing, relative, duplicate, broad, linked, directory, nonregular, or over-limit entries. Real Linux malicious config/filter/fsmonitor/hooks, exact loader/library closure, openat2 topology and races, Landlock/seccomp/rlimit enforcement, process isolation, and packaged runtime remain Slice G `UNVERIFIED`.
Every phase: format check, lint, type check, focused tests, all relevant tests, security suite, `git diff --check`, diff inspection, build/package smoke test, and clean-room validation. Any critical safety test failure blocks continuation. E2E unavailable for a phase is reported `UNVERIFIED`, never passed.

The authoritative deterministic non-production command is `pnpm verify`; it composes add-on mirror, formatting, lint, type, build, and complete test checks.
